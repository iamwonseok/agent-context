# Review Checklist

## Function

- [ ] Meets requirements?
- [ ] All acceptance criteria?
- [ ] No scope creep?
- [ ] Logic correct?
- [ ] Boundary cases?
- [ ] Errors handled?
- [ ] Good error messages?

## Code Quality

- [ ] Easy to read?
- [ ] Clear names?
- [ ] Comments where needed?
- [ ] Single responsibility?
- [ ] No duplication?
- [ ] Right abstraction?

## Performance

- [ ] No unnecessary loops?
- [ ] Right data structures?
- [ ] No N+1 queries?
- [ ] Indexes exist?
- [ ] No memory leaks?
- [ ] Large data handled?

## Security

- [ ] Input validated?
- [ ] SQL injection prevented?
- [ ] XSS prevented?
- [ ] Auth required?
- [ ] Permissions checked?
- [ ] No secrets in logs?
- [ ] No hardcoded secrets?

## Testing

- [ ] New tests added?
- [ ] Existing tests updated?
- [ ] Edge cases tested?
- [ ] Tests meaningful?
- [ ] Tests independent?

## Docs

- [ ] Docstrings on public API?
- [ ] Complex code explained?
- [ ] README updated?
- [ ] API docs updated?

## Infrastructure

- [ ] Config separated?
- [ ] New env vars documented?
- [ ] Migration needed?
- [ ] Rollback possible?
- [ ] Logging added?

## Quick Check

```
[ ] Tests pass?
[ ] Lint clean?
[ ] PR size OK?
[ ] Commit message clear?
[ ] Issue linked?
```
